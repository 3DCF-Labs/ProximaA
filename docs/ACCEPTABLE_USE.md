# Acceptable Use

ProximaA 1.0 is a defensive security tool. Use it only for work you are authorized to do.

## Allowed

- Reviewing code and security findings you have permission to assess.
- Writing safe fixes and regression tests.
- Triage, patch verification, detection engineering, and security education.

## Not allowed

- Writing exploits, malware, or offensive tools.
- Attacking, scanning, or accessing systems without authorization.
- Stealing credentials, hiding activity, or maintaining unauthorized access.
- Anything illegal where you are.

The model is designed to refuse offensive requests. You are responsible for how you use it.
